**Дата:** 2026-06-20
**Статус:** узгоджено в brainstorming; готово до плану реалізації
**Контекст:** інтернет-магазин електрообладнання (React + Vite), натхнений Schneider Electric. Клієнт: Олег Щирба. Розробник: Jeka.

## 1. Проблема та мета

**Проблема.** Стиль «гуляє» і генериться AI-слоп. Причини:
1. У продукті одночасно живуть **3 несумісні естетики** (Tech Precision, Swiss Modernist, Carbon SCADA) + пресети кольорів — кожен екран можна зробити по-різному, тож і робиться по-різному.
2. Згенеровані секції (SCADA-телеметрія, осцилограф, «матеріальний маніфест», лог-консолі) — **не з брифу клієнта**, це AI-наповнення.
3. Свіжий фідбек клієнта (19.06) ще не влитий у спек і місцями суперечить йому.

**Мета.** Зафіксувати **один** дизайнерський візуал як enforceable джерело правди, щоб скіл `frontend-design` (і Claude загалом) **спирався на нього й додавав побажання розробника — без слопу**. Колір/шрифт/тема лишаються **перемиканими токенами** для демонстрації клієнту (стиль/структура — НЕ перемикаються).

**Ключове розрізнення:** «гуляння» було **структурним** (3 розкладки/світи), а не колірним. Лочимо структуру — лишаємо токени крутитися.

## 3. Архітектура рішення: «Style Contract System»

Три частини, які разом не дають слопу будуватись.

### Частина 1 — `STYLE-CONTRACT.md` (огорожа, читається ПЕРШОЮ)
Людино-/агенто-орієнтований документ у корені репо. Токени лишаються в `DESIGN.md`; контракт на нього посилається. Містить:
- **Єдиний стиль** прозою + візуальні принципи (гібрид мінімал+тех).
- **Anti-slop блоклист** (§4).
- **Block recipe** — чек-ліст здачі будь-якого блоку (§4).
- **Accent-пресети** (§5).
- **Перелік дозволених примітивів** + посилання на еталон-екран.

### Частина 2 — Залочені примітиви (код-рівень)
Канонічний набір React-компонентів, які нові блоки **компонують**, а не винаходять із сирого HTML. Менше поверхні для слопу. Перелік — §6.

### Частина 3 — Kitchen-sink еталон-екран `/style`
Одна сторінка, що рендерить усі примітиви + герой + картку товару + сайдбар фільтрів **під живим перемикачем токенів**. Подвійна роль: візуальний якір для розробника/скіла + демо для клієнта.

### Прив'язка enforcement
- Створити **`CLAUDE.md` проєкту** з рядком: *«Перед будь-яким UI/візуальним завданням прочитай `STYLE-CONTRACT.md` і підкоряйся. Не будуй секцій/віджетів, яких не просили.»*
- Додати в `CLAUDE.md` секцію **Verification** з пунктом Playwright-скріншоту (узгоджено з global Feedback Loop розробника).

## 4. Anti-slop блоклист і Block recipe

### Блоклист (жорсткі NEVER)
- ❌ Жодних вигаданих секцій/віджетів, яких не просили (SCADA-телеметрія, осцилографи, «маніфести», лог-консолі, фейкові «живі» дані, лічильники, що тікають).
- ❌ Жодних градієнтів, неон-glow, glassmorphism, важких тіней на контенті — глибина лише тонкими лініями (`--line`).
- ❌ Жодних stock-фото-героїв з overlay-градієнтом.
- ❌ Не змішувати гострі/заокруглені кути в одному в'юпорті.
- ❌ Жодних кольорів поза токенами `DESIGN.md`; зелений — лише за активним accent-пресетом.
- ❌ ALL-CAPS лише для дрібних eyebrow-міток (`label-md`), ніколи для заголовків.
- ❌ Не плодити CTA — один зелений primary на блок/секцію.
- ❌ Без емодзі-іконок, decorative-ілюстрацій, «3D»-рендерів.

### Block recipe (чек-ліст здачі)
Кожен новий блок ПЕРЕД здачею:
1. Використовує лише токени з `DESIGN.md` (кольори/відступи/радіуси/типографіка).
2. Складений із залочених примітивів; новий примітив — лише з явного дозволу розробника.
3. Робить **рівно те, що просили** — жодних «бонусних» секцій.
4. Проходить anti-slop блоклист вище.
5. Рендериться під **обома** accent-пресетами і світлою/темною темою без поломки контрасту (WCAG AA 4.5:1 для тексту).
6. Скріншот через Playwright для візуальної самоперевірки.

## 5. Accent-пресети (зелений↔чорний)

Два перемикані пресети. Клієнт обере на живому еталоні.

**Пресет A — «Гроші / структура»**
- Зелений (`--accent #3DCD58`): кнопка Купити/кошик, статус «в наявності».
- Світлочорний (`--ink #1B1D1F`): eyebrow-мітки, NEW, активні таби, ціна, технічні цифри.

**Пресет B — «Тільки Купити»**
- Зелений: лише головна дія «Купити».
- Світлочорний: усе інше акцентне (ціни, NEW, eyebrow, ховери, активні стани). Зелений рідкісний → потужний; строгіший «дорогий» вигляд.

Технічно — додатковий перемикач у `TweaksPanel`, що змінює мапінг ролей на існуючих токенах (без нових кольорів).

## 6. Залочені примітиви

Аудит поточних компонентів (`src/components/*`) → зведення до канонічного набору:

| Примітив | Призначення |
|---|---|
| `Button` | Варіанти: `primary` (зелений, темний текст), `dark` (світлочорний), `ghost`. Один primary на блок. |
| `Card` | Біла, радіус `lg`, межа `--line`, без важкої тіні. |
| `HeroBand` | Світлочорна смуга `#17181A`, великий sentence-case заголовок, зелена eyebrow, зелений CTA. Точково. |
| `Chip` / `Eyebrow` | Дрібна ВЕРСАЛЬНА мітка (`label-md`, tracking 0.16em). |
| `ProductCard` | Стиль axiom: чиста, великий заголовок, моно-артикул, ціна/кнопка за accent-пресетом, мінімум обведень. |
| `FilterSidebar` | Легкий: згорнуті групи, без важких рамок/фонів/лічильників (антиприклад — schneidershop). |
| `Price` | Ціна за accent-пресетом. |
| `SkuMono` | Артикул/специфікація — IBM Plex Mono (`--mono`). |
| `TechLine` | Тонкий прозоріший роздільник/межа (фідбек клієнта). |

Нові блоки компонують лише ці примітиви.

## 7. Перемикач (демо) — урізання

`TweaksPanel` лишається, але **тільки токени**:
- Палітри (пресети + крутилка відтінків зеленого).
- 4 шрифт-сети (вже є): `unbounded_onest`, `onest` (дефолт), `fixel`, `e_ukraine`.
- Світла/темна тема.
- Accent-пресет (A/B).

**Прибираємо перемикання стилів** — 3 стилі більше не існують у продукті.

---

## 8. Поза межами (YAGNI)

- Не чіпаємо логіку кошика, даних товарів, навігації.
- Не ребрендимо всі екрани цієї ітерації — спершу контракт + примітиви + еталон-екран.
- Код Carbon SCADA / жорсткого Swiss / «слоп»-секцій — видаляємо окремим кроком плану (не лишаємо «про всяк випадок»).
- Не додаємо нові стилі/теми поза обраним візуалом.

---

## 9. Критерії готовності

- `STYLE-CONTRACT.md` існує, на нього посилається `CLAUDE.md` проєкту як «читати першим».
- У продукті лишився **один** візуальний стиль; Carbon/Swiss/слоп-секції видалені.
- `TweaksPanel` перемикає лише токени (палітра/шрифт/тема/accent-пресет), не стилі.
- Еталон-екран `/style` рендерить усі примітиви під живими токенами і проходить anti-slop блоклист.
- Збірка `npm run build` зелена; візуальна перевірка скріншотом.
